% --- Tagged equation environment (fix hyperref duplicate destinations) -----
% amsmath's \tag{...} replaces the visible equation number but rolls the
% equation counter back at \end{equation}, so multiple \tag-labelled
% equations all end up placing their hyperref anchor at the SAME
% counter value (e.g. equation.24), triggering pdfTeX's "destination
% with the same identifier has been already used, duplicate ignored"
% warnings and breaking \ref navigation to the later tagged eqs.
%
% Workaround: use equation* (unnumbered, no counter anchor) and manually
% set \@currentlabel so \ref prints the custom tag. \phantomsection
% creates a unique hyperref destination right before the equation so
% each \label binds to its own fresh anchor. Usage:
%     \begin{taggedeqn}{P}{P}
%         \hat{\boldsymbol{\theta}} = \arg\min ...
%     \end{taggedeqn}
% First arg is the visible tag text, second is the slug for eq:<slug>.
\makeatletter
\newenvironment{taggedeqn}[2]{%
    \gdef\@currentlabel{#1}%
    \phantomsection\label{eq:#2}%
    \begin{equation*}\tag*{(#1)}%
}{%
    \end{equation*}%
}
\makeatother

\begin{abstract}
We estimate Herschel--Bulkley (HB) parameters
$(n,\eta,\sigma_Y)$ from dam-break video observations by replacing
simulation-in-the-loop inversion with a local Gaussian-process (GP)
surrogate trained on curated simulation data. The predecessor protocol of
\cite{hamamichi2023nonnewtonian} repeatedly evaluates a
material point method (MPM) forward simulator inside a derivative-free
optimizer, taking around eight hours per setup. We instead train a
$2{,}775$-expert local GP bank on $407{,}480$ cleaned MPM rows with
fixed density $\rho=1.0$ to emulate
$(n,\eta,\sigma_Y,W,H)\mapsto(x_{01},\ldots,x_{08})$, and propagate
GP posterior variance, local Bayesian approximation error (BAE), and
observation noise into the inverse objective.

The resulting inverse is surrogate-only: it generates or optimizes a
candidate pool under the GP+BAE score and returns either a supported
point estimate or an ambiguous plausible set when a single dam-break
observation does not justify a unique parameter triple. On the
held-out forward test split the GP bank achieves sub-cm trajectory
error, with overall MAE $0.05296$\,cm and $x_{08}$ MAE
$0.08790$\,cm. On a $64$-case synthetic inverse validation, the
plausible set contains a near-truth representative in $98.4$--$100.0\%$
of cases, while forced rank-$1$ selection remains low
($6.3$--$14.1\%$), motivating ambiguity reporting rather than
overconfident single-theta output.

We further report setup-selection diagnostics, speed measurements,
real-material diagnostics, and setup-level held-out validation for
same-session repeated captures. The
unoptimised Python implementation scores a $512$-candidate inverse pass
with plausible-set summaries in $3.63$\,s per case/setup, and one
surrogate candidate/setup evaluation is
$190$--$618\times$ faster than measured one-shot MPM replays. These
results support a fast GP-surrogate inverse with explicit uncertainty
and plausible-set validation. Rheometer-infeasible materials are
evaluated by dam-break replay diagnostics, and time-varying materials
are evaluated at the held-out setup level rather than as a fitted
kinetic ageing law.
\end{abstract}
